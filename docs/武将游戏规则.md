## 4. 当前原型范围

- 当前 web 原型已实现前 5 个武将。
- 当前 web 原型已改为房间制在线测试版：
  - 一位玩家创建房间
  - 另一位玩家通过房间链接或房间码加入
  - 双方在同一房间大厅各自选择自己的武将
  - 开始后进入同一局在线对战
- 当前联机同步采用 HTTP 轮询实现，适合先部署到 Windows 机器上做朋友间测试。
- 当前引擎已使用严格 OOP 结构实现。
- 每个武将都从抽象父类体系向下派生。
- 技能、特性、状态、场地、连锁窗口都已拆入独立类层级。
- 当前战斗页采用单屏紧凑布局：在桌面测试时，战场、行动队列、连锁窗口和战斗记录应尽量同时出现在同一屏内，而不是依赖页面滚动。
- 当前原型交互补充：
  - 进入应用时会先统一确认当前浏览器这位玩家要使用的昵称；之后创建房间、输入房间码加入、以及从已有房间列表直接加入，都会复用这一个昵称，不再分别填写。
  - 昵称可以留空；若留空，系统会自动使用默认昵称参与创建房间和加入房间。
  - 这个昵称现在按“当前窗口 / 当前标签页”保存：同一个窗口刷新页面后仍会保留，但另开一个窗口或标签页时，会重新要求输入昵称，并被视为另一位玩家。
  - 首页会提供“创建房间”和“加入房间”两种入口。
  - 房主现在可以删除自己创建的房间；删除后该房间会立刻从房间列表中消失，当前仍停留在该房间里的其他窗口会在下一次同步时自动退回首页。
  - 进入房间后，现在会额外提供“离开房间”按钮：观战者和大厅中的玩家都可以直接离开并返回首页大厅。
  - 大厅中若普通玩家离开房间，其席位会被释放；若房主离开房间且房间里还有其他玩家，则房主身份会自动转交给仍留在房间里的那位玩家。
  - 若房间里最后一位玩家离开，则该房间会自动关闭并从房间列表中消失。
  - 对局进行中时，已占用席位的玩家不能直接离开房间，以免把房间卡死；需要先投降或等对局结束。
  - 首页大厅会额外显示当前进程里已经存在的房间列表，方便测试时直接加入或返回自己之前进入过的房间。
  - 首页大厅当前按“左侧入房操作 / 右侧已有房间目录 / 下方武将图鉴”布局实现；在常规桌面缩放下，不应再需要把浏览器缩到 50% 才能看到已有房间。
  - 通过邀请链接或大厅里的已有房间进入时，默认不会自动继承之前浏览器里保存的玩家席位；页面会直接使用当前浏览器已经确认好的昵称，作为空余席位加入。
  - 如果当前浏览器确实需要回到自己之前占用过的席位，需要点击单独的“继续原身份”按钮恢复，而不是自动恢复。
  - 点击“加入房间”时，会直接读取当前输入框中的房间码，并复用当前浏览器已经确认好的昵称发起加入，不会把按钮点击事件误当成房间码。
  - 如果加入失败，页面会直接显示后端返回的明确原因，例如“房间不存在”“房间已经满员”或“对局已经开始”，方便联机测试时快速判断问题。
  - 房间大厅会显示房间码、当前席位、邀请链接、双方选将状态与开始按钮。
  - 当房间从大厅进入对战后，已占用席位的浏览器会自动切到战场页；若只是观战视角，则仍停留在大厅，可手动进入战场查看。
  - 当前窗口若已经占用席位，会自动记住该房间的玩家 token；刷新页面后会尝试继续回到原席位，但另开一个窗口不会继承这个席位。
  - 对局结束后，房间内玩家可以直接在终局界面或房间大厅点击“重新开始选将”，让同一房间回到大厅并清空双方本局所选武将，然后重新选将开新一局。
  - 对局进行中，任意一方玩家都可以主动选择“投降”来立刻结束本局，另一方直接获胜。
  - 对局开始后，未轮到你操作时仍可以继续悬停查看战场单位信息。
  - 点击敌方武将会取消当前己方行动选择，并在右侧信息栏显示该敌方单位信息。
  - 点击空格会清空当前信息与行动轮盘。
  - 蓝色高亮表示当前动作真实可选的格子或单位；像魔墙这类可对自己使用的技能，也会把自己标成蓝框。
  - 选择穿刺后，会先标出所有合法可选格；第一次点击会锁定并高亮第 1 格，第二次点击另一个合法格后才会正式施放。
  - 处于无法被选中 / 隐身状态的单位，不会再被标成可直接点选的攻击或技能目标。
  - 鼠标悬停在棋盘上的单位上时，会在鼠标旁边显示该单位的即时信息卡。
  - 棋子本体当前只显示名字；生命改为外围饼形血圈。
  - 魔力当前按“精确显示”实现：魔量 `<= 5` 时直接显示具体几个蓝色魔块；魔量 `> 5` 时显示 1 个蓝色魔块加右侧数值。
  - 空白格不再默认显示坐标。
  - 战场上方会单独显示当前正在生效的战场状态 / 场地效果，例如大圣光、大火葬。
  - 若某个格子正被场地效果持续影响，格子左上角会直接显示对应的小标记；同一格若叠了多个场地效果，也会一起显示。
  - 回避当前按“直线恰好 2 格”实现，中途可穿过单位，但不能拐弯、只走 1 格，或原地留在自己当前格。
  - 遁入黑暗当前按“进入隐身；技能现身只取消隐身，普攻现身则强化那次普攻”实现。
  - 命运飞踢当前按“先冲刺、再掷硬币、若判到目标消失则这段后续效果才开启连锁窗口”实现。
  - 消失中的单位会以可见标记留在棋盘上，但不占格；若回归时原位被占，会先进入选格阶段。
  - 每次动作结算或换边后，界面会自动切到当前可操作单位；若当前在处理重新出现选格，则优先切到该单位。
  - 选中需要目标的行动后，行动轮盘会先收起，避免遮挡棋盘点击；提示卡和战场提示本身也不会拦截鼠标。
  - 选中需要目标的行动时，棋盘下方会额外出现“取消选择”按钮；即使是全场都可选的目标技能，也可以在战场外直接取消。
  - 棋盘选格当前按直接点击高亮格结算；点击无效格会取消本次目标选择。
  - 手机端战斗页当前按“棋盘优先”布局实现：战场不再被右侧信息栏压缩成很小的一团，而是优先保持可点击大小；其余信息面板会堆叠到棋盘下方，通过页面滚动继续查看。
  - 手机端战斗页当前允许纵向滚动整页；悬浮信息卡会隐藏，底部操作按钮会改成纵向整行按钮，方便触屏点按。

## 5. Windows 部署测试

- 当前 `run.py` 仍支持传统的 `0.0.0.0:8000` 监听方式，但现在更推荐直接双击 `start_windows_server.bat`。
- `start_windows_server.bat` 当前默认行为已经改成“公网临时隧道模式”：
  - 双击后会优先尝试启动一个临时公网 HTTPS 链接，供外网朋友直接访问。
  - 这个模式当前通过 Cloudflare Quick Tunnel 提供，仅适合联机测试，不适合正式长期部署。
  - 在这个模式下，本地游戏服务会只绑定 `127.0.0.1`，再通过临时隧道转发到公网，因此不需要你手动开放路由器端口转发。
  - 启动脚本会在本地找 `cloudflared`；若没有找到，会自动下载一份到 `tools/cloudflared/cloudflared.exe`。
  - 启动成功后，脚本会打印并自动打开公网主页链接；房间里复制出的邀请链接也会使用这个公网地址。
  - 这个公网地址是临时的，重启脚本后通常会变化。
- 若你只想做局域网测试或手动指定自己的公网 IP / 域名：
  - 可运行 `powershell -ExecutionPolicy Bypass -File .\scripts\start_windows_server.ps1 -LanOnly`
  - 或直接运行 `python C:\Users\jiz14\TeamGH\wujiang_game\run.py --host 0.0.0.0 --port 8000 --public-base-url http://你的地址:8000`
  - 如果你有域名或反向代理，也可以把 `https://game.example.com` 这类站点根地址直接填进 `--public-base-url`
- 现在仓库根目录提供的 Windows 辅助入口如下：
  - `start_windows_server.bat`：默认启动公网临时隧道模式；若隧道失败，会提示是否退回局域网模式继续启动。
  - `open_firewall_8000.bat`：单独为 TCP 8000 长期开启 Windows 防火墙入站规则；需要以管理员身份运行。
  - `close_temporary_firewall_8000.bat`：若你在局域网模式下使用了临时防火墙规则，且启动窗口被强制关闭，导致规则没来得及自动删掉，可以手动双击这个脚本清理；同样需要以管理员身份运行。
- 在公网临时隧道模式下：
  - 一般不需要手动开放 Windows 防火墙入站 8000，也不需要路由器端口映射。
  - 朋友直接打开脚本给出的公网主页链接即可进入首页。
  - 朋友也可以直接打开房间中的邀请链接；页面会先要求输入昵称，再以新的玩家身份进入房间。
- 在局域网 / 手动公网地址模式下：
  - 若你不填写 `--public-base-url`，房间邀请链接会跟随你当前浏览器访问服务时使用的地址；如果你自己是用 `localhost` 打开的，那么复制给朋友的链接也会变成 `localhost`，对方无法访问。
  - 若朋友不在同一局域网，除了开放本机防火墙端口外，还需要路由器或云主机层面的端口转发 / 映射。
- 当前测试版房间数据保存在进程内存中：
  - 服务重启后，现有房间会消失
  - 适合当前小范围联机测试
  - 还不适合作为长期正式服方案

## 6. 文档维护约定

- 以后每次规则、技能、特性、武将机制发生改动，都必须同步更新这 3 份主文档。
- 文档中的持续时间统一优先使用“轮”。
- 行动次数限制保留“每回合最多使用 X 次”的写法。
- 若某个规则涉及以下底层语义，必须至少同步更新《通用规则》和《武将说明》：
  - 对格子结算
  - 伤害公式
  - 护盾与破魔
  - 召唤物行动时机
  - 召唤者死亡带走召唤物
  - 胜利条件
- 若某个规则涉及以下连锁语义，必须至少同步更新《通用规则》和《通用技能和特性说明》：
  - 是否可以不连锁
  - 护盾是否在连锁前自动挡下
  - 破魔是否仍允许连锁
  - 技能费用是在声明时付还是在结算后付
  - 隐身或无敌是否会让这次动作不再开启连锁窗口
