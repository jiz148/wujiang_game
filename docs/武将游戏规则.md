## 4. 当前原型范围

- 当前 web 原型已实现前 5 个武将。
- 当前引擎已使用严格 OOP 结构实现。
- 每个武将都从抽象父类体系向下派生。
- 技能、特性、状态、场地、连锁窗口都已拆入独立类层级。
- 当前战斗页采用单屏紧凑布局：在桌面测试时，战场、行动队列、连锁窗口和战斗记录应尽量同时出现在同一屏内，而不是依赖页面滚动。
- 当前原型交互补充：
  - 点击敌方武将会取消当前己方行动选择，并在右侧信息栏显示该敌方单位信息。
  - 点击空格会清空当前信息与行动轮盘。
  - 蓝色高亮表示当前动作真实可选的格子或单位；像魔墙这类可对自己使用的技能，也会把自己标成蓝框。
  - 处于无法被选中 / 隐身状态的单位，不会再被标成可直接点选的攻击或技能目标。

## 5. 文档维护约定

- 以后每次规则、技能、特性、武将机制发生改动，都必须同步更新这 3 份主文档。
- 文档中的持续时间统一优先使用“轮”。
- 行动次数限制保留“每回合最多使用 X 次”的写法。
- 若某个规则涉及以下底层语义，必须至少同步更新《通用规则》和《武将说明》：
  - 对格子结算
  - 伤害公式
  - 护盾与破魔
  - 召唤物行动时机
  - 召唤者死亡带走召唤物
  - 回合开始胜利条件
- 若某个规则涉及以下连锁语义，必须至少同步更新《通用规则》和《通用技能和特性说明》：
  - 是否可以不连锁
  - 护盾是否在连锁前自动挡下
  - 破魔是否仍允许连锁
  - 技能费用是在声明时付还是在结算后付
  - 隐身或无敌是否会让这次动作不再开启连锁窗口
