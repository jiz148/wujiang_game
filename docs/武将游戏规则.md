## 4. 当前原型范围

- 当前 web 原型已实现前 5 个武将，并继续逐个接入后续武将；当前后续批次已包含元素猎人、不死王利娜、岩神。
- 当前项目新增了长期沙盒战略层规则文档：[英灵城邦战略模式规则.md](C:/Users/jiz14/TeamGH/wujiang_game/docs/英灵城邦战略模式规则.md)，以及落地计划：[英灵城邦战略模式实现计划.md](C:/Users/jiz14/TeamGH/wujiang_game/docs/英灵城邦战略模式实现计划.md)。
- 规则层面已经按“2 人或多人对战、且一个玩家可操作多个武将”的语义维护。
- 当前 web 房间仍是 2 席位，但 `classic` 模式已经定义为“双人各自选择多个武将”的固定出生模式；`random` 模式继续保留为双人随机选人模式。
- `classic` 模式下：
  - 双方都可以重复选择同一武将。
  - 选将数量不设硬上限；前端需要通过棋盘缩放 / 缩略视图来承接更大的战场。
  - 行动顺序不是“某一整边全部走完”，而是双方按各自排序后的顺位交替行动。
  - 每边的排序规则为：`速` 降序，若相同则 `等级` 降序，再 `攻` 降序，再 `守` 降序，再 `范` 降序，再 `魔` 升序，仍相同则随机。
  - 开局先分别排好双方列表，再由双方各自第 1 名比较决定哪一边先手，然后按顺位交错成一整轮顺序。
  - 开局后不再因为临时能力值变化重算整局顺序；被破坏的武将只是在轮到自己的固定顺位时被跳过。
  - 暂时消失 / 放逐但未被破坏的武将，不按死亡跳过；轮到该顺位时要先处理其回场 / 重新出现，再继续该武将的回合。
  - 召唤物、分身、坐骑不进入独立轮转表，只在所属武将的回合内行动。
  - 棋盘与出生点不再写死为 `8 x 8`。棋盘尺寸要同时考虑双方选将数量，以及这些武将开局实际占格大小；出生点固定、左右对称，并按开局行动顺位分配。
- `random` 模式下：
  - 仍然是双人对战，但由房主设置 `n`，开局时双方各随机获得 `n` 个不重复的武将；同一局场上不会出现相同武将。
  - 地图大小计算方式与 `classic` 相同，行动顺序规则与 `classic` 相同。
  - 与 `classic` 的区别只在出生：`random` 使用随机出生位，而不是固定对称出生位。
- 所有模式下，若从开局开始累计已经完成的武将回合数达到 `20 * 开局总武将数量` 后仍未分出胜负，则立即随机判定 1 方获胜；这里的总武将数量只计算开局参战的非召唤物武将。

### 4.1 已实现的多席位房间规则

- 房间将扩展为 `2~6` 个席位，但仍然只分红 / 蓝两队，不做自由混战。
- 房主在大厅里手动给每个席位分配队伍。
- 每个席位只有 `开放 / 真人 / AI` 三种状态。
- 只有当全部席位都已经被真人或 AI 占满时，房间才能开始。
- 房间至少要保留 `1` 个真人席位，不能整间房全是 AI。
- `classic` 与 `random` 都支持多人席位、真人 / AI 混合，以及手动分队。
- 在多人席位下，行动顺序仍然按“两队交替”，不是按席位交替：
  - 先把红队所有武将合并排序成一张表，蓝队也合并排序成一张表。
  - 然后按 `红1 -> 蓝1 -> 红2 -> 蓝2 -> ...` 交替；若一边顺位先耗尽，另一边继续剩余顺位。
- 同队不同席位之间，各自只主动操作自己所属的武将；但支援型连锁、墙、保护、震开、随时技能等，仍然可以正常支援同队其他席位的武将。
- `random` 模式下，房主设置的是“每队的 `n`”；若一队内有多个席位，则房主还要为该队每个席位设置随机武将配额，且这些配额之和必须等于该队的 `n`。
- `classic` 模式当前允许真人房主作为纯观战席位，不给自己配置武将，只让 AI 席位分别代表红 / 蓝两队出战。

### 4.2 AI 与回放

- AI 使用合法玩家视角，不偷看隐身、分身真假或其他未公开信息。
- AI 难度分为 `简单 / 标准 / 激进` 三档。
- 房间有一个全局默认 AI 难度，同时允许对单个 AI 席位单独覆盖难度。
- 当前房间原型中，AI 已经会主动完成自己席位武将的回合操作，也会处理连锁反应、复活落点，以及对方回合中的随时技能。
- AI 选择攻击或对敌方单位使用技能时，会先假设对方不连锁，并过滤掉无法至少对一个敌方单位造成伤害、消耗护盾 / 资源、或附加有效效果的候选；消耗普通护盾或“保护”等效果留下的临时护盾都算有效影响；多目标或范围动作只要至少有一个敌方单位会受到有效影响，就仍可选择。
- 所有完成的对局都会自动保存回放到项目内的 `replays/` 目录，直到玩家手动删除。
- 回放默认按观战者的合法视角打开；对局结束后允许切换到全知视角。
- 只要战场上已经没有任何真人席位当前仍拥有的武将，房间就会进入 AI 自动推进 / 观战模式。
- AI vs AI 对战当前已经支持实时播放与回放控制，由房主负责控制：
  - `暂停 / 继续`
  - `倍速`
  - `单步前进`
  - `查看历史步骤`
  - `拖动时间轴`
- 当前 web 原型已改为房间制在线测试版：
  - 一位玩家创建房间
  - 另一位玩家通过房间链接或房间码加入
  - 双方在同一房间大厅各自选择自己的武将
  - 开始后进入同一局在线对战
- 当前联机同步采用 HTTP 轮询实现，适合先部署到 Windows 机器上做朋友间测试。
- 当前引擎已使用严格 OOP 结构实现。
- 每个武将都从抽象父类体系向下派生。
- 技能、特性、状态、场地、连锁窗口都已拆入独立类层级。
- 当前战斗页仍优先保持紧凑布局，但桌面端和手机端现在都允许纵向滚动整页；若内容略微超出一屏，会优先通过页面滚动避免显示错位或被裁切。
- 当前原型交互补充：
  - 进入应用时会先统一确认当前浏览器这位玩家要使用的昵称；之后创建房间、输入房间码加入、以及从已有房间列表直接加入，都会复用这一个昵称，不再分别填写。
  - 昵称可以留空；若留空，系统会自动使用默认昵称参与创建房间和加入房间。
  - 这个昵称现在按“当前窗口 / 当前标签页”保存：同一个窗口刷新页面后仍会保留，但另开一个窗口或标签页时，会重新要求输入昵称，并被视为另一位玩家。
  - 首页会提供“创建房间”和“加入房间”两种入口。
  - 房主现在可以删除自己创建的房间；删除后该房间会立刻从房间列表中消失，当前仍停留在该房间里的其他窗口会在下一次同步时自动退回首页。
  - 进入房间后，现在会额外提供“离开房间”按钮：观战者和大厅中的玩家都可以直接离开并返回首页大厅。
  - 大厅中若普通玩家离开房间，其席位会被释放；若房主离开房间且房间里还有其他玩家，则房主身份会自动转交给仍留在房间里的那位玩家。
  - 若房间里最后一位玩家离开，则该房间会自动关闭并从房间列表中消失。
  - 对局进行中时，已占用席位的玩家不能直接离开房间，以免把房间卡死；需要先投降或等对局结束。
  - 首页大厅会额外显示当前进程里已经存在的房间列表，方便测试时直接加入或返回自己之前进入过的房间。
  - 首页大厅当前按“左侧入房操作 / 右侧已有房间目录 / 下方武将图鉴”布局实现；在常规桌面缩放下，不应再需要把浏览器缩到 50% 才能看到已有房间。
  - 通过邀请链接或大厅里的已有房间进入时，默认不会自动继承之前浏览器里保存的玩家席位；页面会直接使用当前浏览器已经确认好的昵称，作为空余席位加入。
  - 如果当前浏览器确实需要回到自己之前占用过的席位，需要点击单独的“继续原身份”按钮恢复，而不是自动恢复。
- 当前窗口第一次进入首页时，会先在页面中央弹出昵称输入层；确认后才进入房间大厅。另开一个窗口或标签页会被视为新的玩家，需要重新输入昵称。
- 点击“加入房间”时，会直接读取当前输入框中的房间码，并复用当前浏览器已经确认好的昵称发起加入，不会把按钮点击事件误当成房间码。
- 如果加入失败，页面会直接显示后端返回的明确原因，例如“房间不存在”“房间已经满员”或“对局已经开始”，方便联机测试时快速判断问题。
- 首页大厅与昵称输入层依赖 `/static/app.js` 正常加载；如果前端资源更新后出现“直接进主页却无法操作”这类现象，需要先强刷页面一次再重试。
- 当前页面允许整页纵向滚动，不再强制锁定为单屏；战斗页会按“最左侧悬浮信息槽 / 中央战场 / 右侧折叠信息栏”排布，左侧悬浮信息槽会固定在战斗页左边，不再把战场本体重新挤窄。
- 左侧悬浮信息只会切换内容显隐，不会再因为鼠标移入移出而重新改动战场布局，也不会拦截棋盘点击；右侧折叠栏展开时，内容也应始终被限制在对应卡片内部，不应再溢出到卡片外。
  - 房间大厅会显示房间码、当前席位、邀请链接、双方选将状态与开始按钮。
  - `classic` 模式大厅里的选将交互按“武将卡 +1 / -1 计数器 + 已选列表”处理，而不是单选 1 个武将。
  - 当房间从大厅进入对战后，已占用席位的浏览器会自动切到战场页；若只是观战视角，则仍停留在大厅，可手动进入战场查看。
  - 当前窗口若已经占用席位，会自动记住该房间的玩家 token；刷新页面后会尝试继续回到原席位，但另开一个窗口不会继承这个席位。
  - 对局结束后，房间内玩家可以直接在终局界面或房间大厅点击“重新开始选将”，让同一房间回到大厅并清空双方本局所选武将，然后重新选将开新一局。
  - 对局进行中，任意一方玩家都可以主动选择“投降”来立刻结束本局，另一方直接获胜。
  - 对局开始后，未轮到你操作时仍可以继续悬停查看战场单位信息。
  - 点击敌方武将会取消当前己方行动选择，并在右侧信息栏显示该敌方单位信息。
  - 点击空格会清空当前信息与行动轮盘。
  - 蓝色高亮表示当前动作真实可选的格子或单位；像魔墙这类可对自己使用的技能，也会把自己标成蓝框。
  - 选择穿刺、机枪、龙息等范围施放技能后，会先标出所有合法可选格；玩家逐格点击形成合法区域，再点击“完成选择”结算，或点击“取消选择”退出。
  - 范围施放技能贴着棋盘边界时，若完整形状的一部分本应落在棋盘外，则只需要选择棋盘内实际存在的那部分格子。
  - 不规则多格单位会显示成一个覆盖全部占用格的整体棋子，内部按实际占用格呈现形状，不显示成多个独立棋子。
  - 处于无法被选中 / 隐身状态的单位，不会再被标成可直接点选的攻击或技能目标。
  - 鼠标悬停在棋盘上的单位或动作上时，会在战场左侧显示临时信息框；不 hover 时会自动消失，不会遮挡棋盘。
  - 右侧信息区当前改成可点击展开的折叠栏，不会再默认把“战斗记录 / 当前信息 / 行动与连锁”三块全部一直展开。
  - 棋子本体当前只显示名字；生命改为外围饼形血圈。
  - 魔力当前按“精确显示”实现：魔量 `<= 5` 时直接显示具体几个蓝色魔块；魔量 `> 5` 时显示 1 个蓝色魔块加右侧数值。
  - 若 `classic` 模式因为多武将和大占格单位导致棋盘显著变大，前端需要提供战场缩放能力，避免大棋盘无法完整查看和操作。
  - 空白格不再默认显示坐标。
  - 战场上方会单独显示当前正在生效的战场状态 / 场地效果，例如大圣光、大火葬。
  - 若某个格子正被场地效果持续影响，格子左上角会直接显示对应的小标记；同一格若叠了多个场地效果，也会一起显示。
  - 回避当前按“直线恰好 2 格”实现，中途可穿过单位，但不能拐弯、只走 1 格，或原地留在自己当前格。
  - 遁入黑暗当前按“进入隐身并保留无法回复；技能会消耗现身机会，普攻现身则强化那次普攻；沙尘只移除隐身部分”实现。
  - 若当前场上的所有武将都同时处于隐身，则会立刻自动解除所有在场武将的隐身。
  - 命运飞踢当前按“先冲刺、再掷硬币、若判到目标消失则这段后续效果才开启连锁窗口”实现。
  - 连锁窗口当前会直接显示原动作本次的具体效果摘要，并统一按“【动作名】：效果”展示；像普攻这类动作会直接带出当前攻值，方便响应方判断是否要连锁。
  - 魔力牵引当前按“只禁止目标下次行动中的常规移动，不禁止飞跃、命运飞踢、回避等其他位移技能或位移效果”实现。
  - 消失中的单位会以可见标记留在棋盘上，但不占格；若回归时原位被占，会先进入选格阶段。
  - 每次动作结算或换边后，界面会自动切到当前可操作单位；若当前在处理重新出现选格，则优先切到该单位。
  - 选中需要目标的行动后，行动轮盘会先收起，避免遮挡棋盘点击；提示卡和战场提示本身也不会拦截鼠标。
  - 选中需要目标的行动时，棋盘下方会额外出现“取消选择”按钮；即使是全场都可选的目标技能，也可以在战场外直接取消。
  - 棋盘选格当前按直接点击高亮格结算；点击无效格会取消本次目标选择。
  - 手机端战斗页当前按“棋盘优先”布局实现：战场不再被右侧信息栏压缩成很小的一团，而是优先保持可点击大小；其余信息面板会堆叠到棋盘下方，通过页面滚动继续查看。
  - 手机端战斗页当前允许纵向滚动整页；桌面端战斗页也同样允许页面向下滚动，以减少锁定单屏时出现的显示 bug。
  - 手机端底部悬浮信息框也会保持为不遮挡棋盘的下方块状区域；底部操作按钮会改成纵向整行按钮，方便触屏点按。

## 5. Windows 部署测试

- 当前 `run.py` 仍支持传统的 `0.0.0.0:8000` 监听方式，但现在更推荐直接双击 `start_windows_server.bat`。
- `start_windows_server.bat` 当前默认行为已经改成“公网临时隧道模式”：
  - 双击后会优先尝试启动一个临时公网 HTTPS 链接，供外网朋友直接访问。
  - 这个模式当前通过 Cloudflare Quick Tunnel 提供，仅适合联机测试，不适合正式长期部署。
  - 在这个模式下，本地游戏服务会只绑定 `127.0.0.1`，再通过临时隧道转发到公网，因此不需要你手动开放路由器端口转发。
  - 启动脚本会在本地找 `cloudflared`；若没有找到，会自动下载一份到 `tools/cloudflared/cloudflared.exe`。
  - 启动成功后，脚本会打印并自动打开公网主页链接；房间里复制出的邀请链接也会使用这个公网地址。
  - 这个公网地址是临时的，重启脚本后通常会变化。
- 若你只想做局域网测试或手动指定自己的公网 IP / 域名：
  - 可运行 `powershell -ExecutionPolicy Bypass -File .\scripts\start_windows_server.ps1 -LanOnly`
  - 或直接运行 `python C:\Users\jiz14\TeamGH\wujiang_game\run.py --host 0.0.0.0 --port 8000 --public-base-url http://你的地址:8000`
  - 如果你有域名或反向代理，也可以把 `https://game.example.com` 这类站点根地址直接填进 `--public-base-url`
- 现在仓库根目录提供的 Windows 辅助入口如下：
  - `start_windows_server.bat`：默认启动公网临时隧道模式；若隧道失败，会提示是否退回局域网模式继续启动。
  - `open_firewall_8000.bat`：单独为 TCP 8000 长期开启 Windows 防火墙入站规则；需要以管理员身份运行。
  - `close_temporary_firewall_8000.bat`：若你在局域网模式下使用了临时防火墙规则，且启动窗口被强制关闭，导致规则没来得及自动删掉，可以手动双击这个脚本清理；同样需要以管理员身份运行。
- 在公网临时隧道模式下：
  - 一般不需要手动开放 Windows 防火墙入站 8000，也不需要路由器端口映射。
  - 朋友直接打开脚本给出的公网主页链接即可进入首页。
  - 朋友也可以直接打开房间中的邀请链接；页面会先要求输入昵称，再以新的玩家身份进入房间。
  - 启动脚本会把当前公网主页链接复制到剪贴板，方便你直接发给朋友。
  - `trycloudflare.com` 链接是临时的；只要你重新启动脚本，或者 `cloudflared` 断线后重连，旧链接就可能失效。
  - 如果朋友打开旧的 `trycloudflare.com` 链接看到 Cloudflare `Error 1033`，通常不是游戏页面坏了，而是旧隧道地址已经作废；需要把脚本当前打印出来的新链接重新发给对方。
- 在局域网 / 手动公网地址模式下：
  - 若你不填写 `--public-base-url`，房间邀请链接会跟随你当前浏览器访问服务时使用的地址；如果你自己是用 `localhost` 打开的，那么复制给朋友的链接也会变成 `localhost`，对方无法访问。
  - 若朋友不在同一局域网，除了开放本机防火墙端口外，还需要路由器或云主机层面的端口转发 / 映射。
- 当前测试版房间数据保存在进程内存中：
  - 服务重启后，现有房间会消失
  - 适合当前小范围联机测试
  - 还不适合作为长期正式服方案
- 当前账号系统只提供 SQLite 注册/登录/session 基础能力，默认数据库路径是 `var/wujiang.sqlite3`，可通过 `WUJIANG_AUTH_DB` 覆盖。现有房间 token 流程仍然保留，后续战略模式存档再绑定到账号。
- 现在所有现有房间模式都必须先登录账号才能创建或加入房间；房间 token 只负责恢复具体席位，不再替代账号登录。
- 英灵城邦战略模式按多人房间制战役设计：房主创建战役大厅后会生成加入码，其他登录账号可在锁定前加入；房主锁定初始玩家后，战役可 resume，但只有曾经参加过的初始玩家可恢复进入，并且必须所有初始玩家都在线后才能继续推进。
- 战略模式允许一个账号拥有多个战役存档。城市兵力进入格子战时需要按城市特色和“战术”科技树规则，以一定比例转化为特色士兵单位。玩家可见战斗触发时由玩家选择手动、AI 自动或观看 AI 战斗。
