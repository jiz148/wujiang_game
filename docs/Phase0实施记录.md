# Phase 0 实施记录

> 当前状态：已结束（2026-07-15，项目负责人豁免真实玩家验证）  
> 阶段目标：新玩家不查规则文档，也能在 3 分钟内完成第一次有效行动，并独立完成一场教学战斗。

## P0.1 最小数据闭环

本地事件写入 `var/analytics.sqlite3`。开发环境可直接打开 `/analytics.html` 查看聚合指标和漏斗，也可通过 `/api/analytics/funnel` 读取原始聚合 JSON。埋点失败不得阻塞玩家操作。

| 事件 | 触发时机 | 允许属性 |
| --- | --- | --- |
| `home_view` | 首页初始化完成 | `entry_state` |
| `quick_start_click` | 点击快速开始 | `entry_state` |
| `tutorial_start` | 固定教学创建成功 | `tutorial_id` |
| `tutorial_step` | 教学步骤开始、完成、重试或跳过 | `tutorial_id`, `step_id`, `status`, `duration_ms` |
| `tutorial_complete` | 教学胜负流程完成 | `tutorial_id`, `duration_ms` |
| `tutorial_exit` | 中途退出教学 | `tutorial_id`, `step_id`, `reason`, `duration_ms` |
| `first_effective_action` | 首次成功移动、普攻或主动技能 | `tutorial_id`, `action_type`, `duration_ms` |
| `match_start` | 对局正式开始 | `match_id`, `mode` |
| `match_end` | 对局正常结束或投降结束 | `match_id`, `mode`, `result`, `duration_ms` |
| `invalid_action` | 后端拒绝玩家操作 | `match_id`, `mode`, `action_type`, `reason` |
| `action_succeeded` | 后端成功接受并结算玩家操作 | `match_id`, `mode`, `action_type` |

服务端只接受上述事件和属性白名单；密码、登录 session token、房间 token及任意额外字段不会写入分析库。匿名会话 ID 仅用于本地漏斗去重，不作为账号身份。

漏斗 `metrics` 直接输出首次有效行动中位时间、教学完成率、对局完成率、非法操作率、教学/对局时长中位数和动作尝试数。

后续 P0 交付段会把快速开始、教学步骤、首次有效行动、对局结束和非法操作接入这些已定义事件。

## P0.2 首页与进入路径

首页默认只展示三个入口，优先级为“快速开始 / 英灵城邦 / 自定义对战”。账号表单、房间码、战役参数和完整房间目录只在玩家选择对应流程后展示。

- 快速开始创建固定的“火葬 vs 艾莉”人机战斗，不要求玩家先配置房间或选将。
- 所有可玩模式统一要求登录；昵称只是游戏内显示名，不能替代账号身份；房间席位凭据只用于恢复已占有的席位。
- 未登录点击任一模式会进入同一个账号门槛并说明原因。
- 快速开始创建的房间公开标记为 `experience_kind=tutorial`，便于教学 UI、埋点和后续结算区分普通房间。

## P0.3 新手武将池与推荐阵容

首次选人默认只展示 10 名新手武将，完整公开武将库通过“查看完整武将库”显式展开。每名新手武将都有定位、难度、主要机制、推荐队友和一句话玩法说明。

新手池：火葬者、精兵、吟游诗人、天位骑士·政宗、销魂的死灵、龙骑、诚实天使欧内斯特、丰收之神·索拉、湿地之主·缇娜、E·暗人。

三套可一键应用的 2 人推荐阵容：

1. 稳健前线：政宗 + 吟游诗人。
2. 远程压制：精兵 + 索拉。
3. 机动突击：火葬者 + 销魂的死灵。

配置源位于 `src/wujiang/web/onboarding.py`；首页英雄接口返回同一份新手数据，推荐阵容由服务端一次性替换当前可编辑席位的阵容，避免客户端半途失败形成残缺配置。

## P0.4 固定教学规则

完整步骤、单步限制、失败恢复、断线恢复、检查点和验收场景已写入《新手教学设计》。教学顺序为：选中火葬者 → 指定移动 → 普攻艾莉 → 使用穿刺 → 结束回合 → 完成一次连锁响应 → 独立达成胜利目标。

核心恢复原则：普通错误原地重试；刷新后由服务端恢复步骤；自由战斗失败回到连锁教学后的检查点；状态损坏时同时提供重新开始和返回首页，不能留下无操作出口。

## P0.5 教学实现

- 服务端拥有教学步骤状态并随房间序列化，刷新和短暂断线后可恢复，不依赖单页内存。
- 前六步由服务端限制允许动作；前端同步隐藏无关行动，并把移动与穿刺预览收窄到固定目标。
- 混合人机教学会自动推进 AI，直到再次需要玩家输入；玩家必须亲手完成一次连锁响应或明确放弃。
- 完成连锁后保存自由战斗检查点；失败可以恢复检查点，胜利结算提供再次开始教学与返回路径。
- 教学面板支持回顾上一步说明、重试、折叠已理解说明和退出；回顾说明不会伪装成战局回滚。
- 主动退出后，首页快速开始会用浏览器保存的最近教学房间编号和既有席位凭据向服务端检查恢复资格；可恢复时同时提供继续与重新开始，失效或已完成时清除入口并回退为重新开始。
- 教学步骤、首次有效行动、非法操作、完成、退出和对局结束已接入 P0.1 事件白名单。

定向行为测试覆盖：固定教学创建、错误步骤拒绝、选中与指定移动、首次行动时间、刷新恢复、错误动作原地阻止、普攻/穿刺/结束回合/连锁推进以及检查点恢复。

## P0.6 行动预览与非法原因

选择行动后，侧栏预测区同步显示资源消耗、合法目标、预计伤害或技能效果、最终站位和当前范围可能影响的单位。普攻使用正式摘要伤害公式给出基础估算，并明确护盾、连锁、免疫和多格命中会改变实际结算；技能无法可靠数值化时展示正式技能效果说明。

移动路径会随逐格选择更新最终落点。教学移动和穿刺的前端预览被收窄到服务端允许的固定格子，实际结算仍由后端权威判定。

点击非高亮格不再静默取消行动：移动会说明速度、阻挡和完整身形边界，普攻会说明范围/直线/空格，技能会说明目标、范围、方向或形状不合法。后端拒绝原因仍原样显示并计入 `invalid_action`。

## P0.7 双端走查与内测准备

实际浏览器走查覆盖桌面首页、账号门槛、固定教学创建、教学选中、移动预览、最终站位和首次行动推进；手机覆盖首页入口、教学首屏、棋盘滚动和触控尺寸。

补充恢复走查覆盖主动退出教学、首页继续/重新开始分流和服务端进度恢复：手机视口下退出房间 `X3HS69` 后，首页识别到未完成的“步骤 1/7：选中你的武将”；点击“继续未完成教学”后回到同一房间、同一步骤。聚合内测看板也已分别在桌面和手机视口验证加载、空状态、刷新入口与响应式布局，浏览器控制台无错误或警告。

走查修复了四项问题：教学首步被状态同步预选；手机页面整体横向溢出；手机先显示长武将详情而非教学目标；深色教学面板文字对比度不足。当前手机页面宽度与视口一致，宽棋盘只在 `board-stage` 内滚动，战场排在详情和战报之前。
