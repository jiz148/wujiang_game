# 武将产品开发路线图

> 文档状态：项目级产品开发优先级的正式依据
> 当前阶段：**Phase 4 — 军队、行军、补给与围城（P4.3 已完成）**
> 生效日期：2026-07-15
> 阶段切换：必须满足退出门槛，并由项目负责人明确确认

## 1. 文档定位

本文决定项目“先做什么、后做什么、暂时不做什么”。

- 具体战斗规则仍以《通用规则》《通用技能和特性说明》《武将说明》及用户最新澄清为准。
- 英灵城邦的具体规则仍以《英灵城邦战略模式规则》为准。
- 英灵城邦已有实现、缺口和技术拆分仍记录在《英灵城邦战略模式实现计划》中。
- 当多个需求同时存在时，除非用户明确改变优先级，否则先完成本文当前 Phase 的目标和退出门槛。
- 修复数据损坏、安全、无法启动、严重规则错误和阻塞测试的问题，不受 Phase 限制。

## 2. 产品定位与发行顺序

### 2.1 长期产品定位

《武将》是一款多人英灵战略游戏：玩家在英灵城邦中经营、任职、结盟和战争，关键冲突进入强调站位、连锁和武将能力组合的格子战斗。

### 2.2 发行顺序

长期定位以英灵城邦为主。2026-07-17 起，开发和验证顺序调整为：

1. 保留已经完成的教学、可重复对战、结算、重连和账号熟练度基础。
2. 立即把英灵城邦推进为一条有时限、有目标、有结局的短战役。
3. 依次补齐中立政治、真实军队与后勤、世界危机、圣物与英灵战略，使每层都形成可玩的闭环。
4. 战役核心成立后，再完成多人异步推进、长期存档、复玩、平衡和正式运营。

武将对战同时承担新手教学、练习、竞技、战斗规则验证和战略战斗承接五项职责。

## 3. 阶段执行规则

每次开发开始前必须标明它属于哪个 Phase，以及它帮助达成哪个目标或退出指标。

1. **当前阶段优先**：Phase 1、插入的 Phase 1.5、Phase 2 与 Phase 3 已由负责人确认结束。2026-07-21 起，当前新增功能默认服务 Phase 4“军队、行军、补给与围城”。
2. **不可按日历自动切换**：文中的周数只是工作量估算，不是切换依据。
3. **阶段退出需确认**：完成退出门槛后，先提交阶段复盘；只有项目负责人确认后才能把下一 Phase 标记为进行中。
4. **不跨阶段堆功能**：后续需求先写入对应 Phase 的候选清单，不因实现简单就提前开发。
5. **规则与稳定性例外**：严重 Bug、规则一致性、存档安全、安全漏洞和阻塞当前工作的基础重构可立即处理。
6. **玩家价值优先**：不能只以“接口存在”或“页面出现”为完成；必须覆盖玩家路径、反馈、错误状态、行为测试和验收指标。
7. **数据不足不得假装达标**：需要真实玩家数据的指标，在埋点和测试样本不足时标记为“待验证”，不能直接进入下一 Phase。

## 4. 全阶段核心指标

| 指标 | 定义 | 目标用途 |
| --- | --- | --- |
| 首次有效行动时间 | 新玩家打开游戏到第一次成功移动、普攻或使用技能的时间 | 判断入口和教学摩擦 |
| 教学完成率 | 开始教学并完成整场教学的玩家比例 | 判断新手体验 |
| 对局完成率 | 已开始且正常完成、投降完成的对局占比；掉线废局单独统计 | 判断核心循环稳定性 |
| 再来一局率 | 完成对局后 10 分钟内开始下一局的玩家比例 | 判断核心乐趣 |
| 对局时长 | 从正式开局到胜负结算的中位数和 P90 | 判断节奏 |
| 回合等待时间 | 玩家获得操作权到提交首个有效操作的耗时 | 判断理解与等待问题 |
| 非法操作率 | 非法目标、非法路径、资源不足等被后端拒绝的操作占比 | 判断规则可读性 |
| 武将使用数据 | 选取率、胜率、阵亡率、技能使用率、组合胜率 | 判断平衡与内容价值 |
| 战役漏斗 | 创建、锁定、进入、推进月份、触发战斗、完成战役的转化 | 判断战略模式完整度 |
| 战役完成率 | 已开始战役中进入正式结算的比例；归档、放弃和损坏分开统计 | 判断大战役是否真正闭环 |
| 月度流失点 | 最后活跃月份与当月未处理阻塞 | 定位节奏、理解和等待问题 |
| 战役时长 | 从锁定战役到正式结算的活跃时长与自然时长 | 判断内容密度与多人摩擦 |
| 决策密度 | 每月高后果决定数、军令使用与计划替换率 | 判断选择是否清晰而非繁琐 |
| 路线分布 | 征服、和平影响、世界主线、圣物等路线的使用与完成 | 判断是否存在有效多解 |
| 格子战密度 | 每场战役真实格子战数量、触发原因和退出率 | 判断双层玩法是否协调 |

## 5. Phase 0 — 第一次好玩

**状态：已结束（负责人豁免真实玩家验证）**
**工作量估算：约 2 周**
**阶段目标：新玩家不查规则文档，也能在 3 分钟内完成第一次有效行动，并独立完成一场教学战斗。**

### 5.1 必做范围

#### A. 首页与进入路径

- 把首页主入口收敛为“快速开始 / 英灵城邦 / 自定义对战”。
- 默认主按钮为“快速开始”，账号、房间码和战役参数进入对应流程后再展示。
- 统一“昵称、临时身份、登录要求”的产品文案和实际规则，消除相互矛盾。
- 对未登录、加载中、服务错误、空房间和恢复旧身份分别提供明确状态。

#### B. 固定新手教学

- 提供固定阵容、固定地图和可重复进入的新手战斗。
- 分阶段教学：选中单位、移动、普攻、主动技能、连锁响应、结束回合、胜负目标。
- 每一步只暴露当前需要的操作，并允许返回、重试和跳过已理解的说明。
- 教学失败时给出原因和恢复路径，不强迫玩家重新开始整局。

#### C. 新手武将池

- 从当前公开武将中选出 8～12 名核心武将作为首次体验池。
- 为核心武将补充定位、难度、主要机制、推荐队友和一句话玩法说明。
- 至少提供 3 套推荐阵容，让玩家可以不读完整武将库直接开局。
- 完整 59 名公开武将继续保留，但不在首次选人路径中平铺展示。

#### D. 战斗可读性

- 选择行动后展示资源消耗、合法目标、预计伤害或效果、最终站位和可能影响的单位。
- 非法格子或目标必须说明具体原因，而不是只取消选择。
- 明确显示当前行动单位、当前阶段、胜负目标和下一步推荐操作。
- 保留后端权威判定；前端预览必须与实际结算一致，并覆盖多格单位和连锁窗口。

#### E. 最小数据闭环

- 定义并记录：进入首页、点击快速开始、开始教学、教学步骤、教学完成/退出、首次有效行动、对局开始/结束、非法操作。
- 本地开发环境能够查看事件和关键漏斗；正式第三方分析工具可在后续确定。
- 埋点不得记录密码、session token、房间 token 或不必要的个人信息。

#### F. 质量与走查

- 覆盖桌面端和手机端核心路径。
- 为首页流程、教学步骤、预览/实际结算一致性和错误状态增加行为测试。
- 保持完整自动化测试通过。

### 5.2 Phase 0 暂缓项

- 不大规模实现剩余 Excel 武将。
- 不扩大战略科技树、外交、圣物、祭坛和更多职位系统。
- 不开发排位赛、赛季、付费成长或复杂商业化。
- 不把新的大页面继续无边界堆入单体前端文件；为教学和新入口先划清模块边界。

### 5.3 退出门槛

- 至少一条从首页进入、完成教学、查看结算并再次开始的完整路径可用。
- 内部测试中，首次有效行动中位时间小于 3 分钟。
- 至少 70% 的有效测试玩家能够完成教学；样本不足时保持在 Phase 0。
- 测试玩家无需打开外部规则文档即可完成一场 AI 教学对局。
- 教学与快速开始的关键事件均可统计。
- 所有相关行为测试和完整测试通过。
- 项目负责人确认 Phase 0 复盘后，才能进入 Phase 1。

## 6. Phase 1 — 可重复对战闭环

**状态：已结束（负责人确认）**
**工作量估算：约 3～6 周**
**阶段目标：玩家完成第一局后，能够低摩擦地再开一局，并获得清晰的局内和赛后反馈。**

### 6.1 计划范围

- 增加快速 AI 对战，并根据新手武将池提供默认阵容。
- 完善武将定位、难度、筛选、搜索、推荐阵容和队伍合法性反馈。
- 增加房间准备状态、开局确认、回合计时、掉线提示和可靠的重连体验。
- 制作赛后总结：胜负原因、伤害、治疗、承伤、连锁、关键回合和 MVP。
- 提供“一键再来一局”、返回选将、最近战绩和最近回放入口。
- 强化攻击、护盾、连锁、死亡、胜负的视觉和声音反馈。
- 完成键盘操作、可见焦点、色弱方案、减少动态效果和触屏目标尺寸检查。
- 拆分首页、房间、战斗、回放等前端职责，降低继续迭代的回归风险。

### 6.2 退出门槛

- 对局完成率达到 75% 以上。
- 标准测试局中位时长控制在 15～25 分钟，P90 有明确监控。
- 新玩家完成 3 局后，非法操作率低于 5%。
- 至少 30% 的完成玩家在 10 分钟内开始下一局。
- 断线后能够恢复正确身份、战场和当前操作状态。
- 桌面和手机端都能完成创建/加入、选将、战斗、结算、再来一局的完整流程。
- 项目负责人确认 Phase 1 复盘后，才能进入 Phase 2。

## 6.5 Phase 1.5 — 中立城邦前置机制

**状态：已完成（负责人已确认进入后续阶段）**
**阶段目标：让新战略地图以中立小势力为主体，形成可经营、可征服、可被外交利用的缓冲地带。**

### 6.5.1 规则与范围

- 新战役中每个玩家或主要 AI 势力只拥有一座起始城市；其余城市各自组成“一城一邦”的中立城邦。
- 中立城邦数量必须严格多于玩家与主要 AI 的起始城市总数；不满足时拒绝生成该地图规模。
- 每个中立城邦是独立小势力，拥有具名城主、城市资源、守军和城防；城主负责本城的基础经营与防守。
- 中立城邦默认不主动攻击玩家或主要 AI，也不执行主要势力的扩张、科技、祭祀和故事决策链。
- 主要势力的主公可消耗 1 点军令和 60 金钱，教唆一个与目标势力接壤的中立城邦攻击该目标势力。
- 教唆为一次性指令：中立城邦在下次月度 AI 行动中选择一个合法相邻目标发动快速战斗，出兵后解除；目标不再合法时保留教唆，等待出现合法相邻目标。
- “消灭敌对势力”只比较主要势力；中立城邦不阻止该目标达成。“统一城邦”仍要求控制地图上的全部城市。

### 6.5.2 完成门槛

- 新地图稳定生成且中立城邦数量规则成立，每个中立城市都有可见城主与守军。
- 未受教唆的中立城邦跨月不会主动进攻；受教唆且存在合法相邻目标时只发动一次进攻。
- 战略地图能明确区分己方、主要敌方与中立城邦，并显示城主、被教唆状态和合法教唆入口。
- 规则、存档兼容、行为测试与项目地图同步完成。

## 7. Phase 2 — 战役闭环与节奏

**状态：已结束（负责人确认）**
**建议拆分：P2.1～P2.6**
**阶段目标：一名真人与 AI 能在 60～90 分钟内完成一场 8 城、12 个月的大战役，始终知道当前目标、本月关键决定和战役何时结束。**

### 7.1 玩家旅程

创建 8 城战役 → 选择唯一战略武将与身份 → 完成前三个月引导目标 → 经营、祭祀、处理事件并参与至少一次城市冲突 → 在第 12 月或提前胜利时进入正式结算 → 选择结束或继续沙盒。

### 7.2 计划范围

1. **P2.1 战役契约（已完成，2026-07-17）**
   - 固定首个验证场景：2 个主要势力、6 个中立城邦、12 个战略月。
   - 新建战役时明确显示预计时长、城市构成、已开放胜利方式和未开放系统。
   - 保留统一/消灭的提前胜利；增加第 12 月战役评议，按城市控制、城市民心、势力存续、关键战斗与主线进度生成结果，而非强迫统一全图。
2. **P2.2 月初—月末信息闭环（已完成，2026-07-17）**
   - 月初显示“上月发生了什么 / 本月必须处理什么 / 推进后预计发生什么”。
   - 为粮食、维护、民心、叛乱、建筑和行动队列提供月结预测；不泄露战争随机结果。
   - 行动提交后明确显示被替换的计划、剩余军令、资源余额和受影响的未来月份。
3. **P2.3 前三个月引导（已完成，2026-07-18）**
   - 用情境目标引导玩家完成查看边境、设置方针、处理事件、祭祀或任命、准备一次冲突。
   - 引导只限制首次场景，不改变普通战役规则；允许有说明地跳过。
4. **P2.4 官职自动化与决策降噪（已完成，2026-07-18）**
   - 保留四级职位与权限；常规维护使用默认方针或职位 AI 自动执行。
   - 玩家每月优先处理 1～3 个高后果决定，命令和请求必须反馈执行者、成本、预计完成时间与结果。
5. **P2.5 终局与战役复盘（已完成，2026-07-18）**
   - 增加明确的战役完成状态、胜利/败北/存续结局、关键月份、城市变化、格子战结果和角色经历。
   - 结算后可结束归档或继续沙盒；继续不反复发放同一结局。
6. **P2.6 AI 基础目标（已完成，2026-07-18）**
   - AI 每 2～3 个月选择一个可见战略目标，例如稳住粮荒、守住边境、祭祀扩军或夺取某城。
   - AI 仍由地理、资源和威胁决定行为，不加入固定性格模板。

### 7.3 退出门槛

- 一名真人与 AI 可以从创建战役一直完成第 12 月结算，无需修改数据库或手工修复状态。
- 玩家在任意月份都能从主界面回答：战役剩余多久、本月最重要的风险是什么、推进月份会结算哪些内容。
- 至少一次完整战役包含身份选择、城市治理、祭祀或任命、事件处理、城市冲突和最终复盘。
- 统一/消灭提前胜利与限时评议都只结算一次，并允许明确选择继续沙盒。
- 创建、锁定、进入、月 3、月 6、月 9、月 12、战斗触发和战役完成均有漏斗事件。
- 相关行为测试与完整测试通过；项目负责人收到 Phase 2 复盘并明确确认后，才能进入 Phase 3。

## 8. Phase 3 — 中立城邦政治、外交与占领治理

**状态：实现完成（等待项目负责人确认）**
**阶段目标：让地图上数量最多的中立城邦成为可交涉、可利用、可保护、可整合的小势力，而不是等待被吞并的资源点。**

### 8.1 计划范围

- 每个中立城邦拥有可见的关系、姿态、当前诉求、恐惧来源与城主立场；不增加固定“性格”，立场由城市处境生成。
- 提供援助、贸易、保护、索贡、互不侵犯、威慑、教唆和和平整合的最小行动集。
- 关系变化必须有成本、冷却、期限、信誉与世界记忆；承诺失败会影响后续谈判。
- 影响力与支持度进入战役评议，使和平路线能够与直接征服竞争。
- 城市被占领后必须选择自治、整合、驻军或掠夺；选择改变收入、兵力、民心、叛乱和复国风险。
- 叛乱增加谈判、自治、倒戈与外部资助分支；保留现有安抚、赈济、镇压和清剿。
- 主要 AI 与中立城主使用同一外交合法性、成本和记忆规则。

### 8.2 退出门槛

- 每个中立城邦至少存在两条实际可行的处理路线，其中一条不要求攻城。
- 一场完整测试战役中，和平影响/保护与武力征服都能带来可比较的战略收益。
- 违约、教唆、保护失败和占领政策会在后续月份产生可见且可追溯的后果。
- 新占城市不会在战斗结束后无成本稳定；玩家能预判四种占领政策的短期与长期影响。
- AI 能发起、接受或拒绝最小外交行动，并能基于关系与威胁改变计划。
- 项目负责人确认 Phase 3 复盘后，才能进入 Phase 4。

## 9. Phase 4 — 军队、行军、补给与围城

**状态：进行中（P4.3 已完成，等待继续 P4.4）**
**阶段目标：让路线、时间、补给、指挥链和战前准备决定战争结果，消除“相邻城市瞬移进战斗”。**

### 9.1 计划范围

- 实现持久战略军队实体：所属势力、位置、指挥官、兵员、特色单位、粮草、士气、状态与当前命令。
- 把城市征兵、注册单位、大将军分配、将军库存串成“编军—补给—出征”的真实链路。
- 行军按路线占用战略时间；支持集结、巡逻、增援、拦截、撤退和追击。
- 粮草由城市供给并受距离、路线和敌情影响；断粮削弱军队而不是只扣城市数字。
- 攻城进入持续状态，包含围困、强攻、等待、突围、援军和投降；城市不会在宣战当月自动易主。
- 格子战读取军队兵员、补给、士气、城防、地形与援军条件，并完整写回幸存、伤兵、撤退、英灵和占领状态。
- 地图与战情室显示军队路径、预计到达月、补给风险和可能遭遇。

### 9.2 退出门槛

- “编军 → 行军 → 遭遇/围城 → 格子战或快速结算 → 撤退/占领 → 战后治理”全链可完成。
- 同一支军队跨月保持身份、单位库存、粮草、指挥官和损失，不从城市重复生成。
- 至少存在一个因路线/补给/增援选择而改变胜负的行为测试场景。
- 官职链对战争产生正向能力和协作价值，而不只是增加操作门槛。
- 快速结算和真实格子战使用相同战略输入并返回兼容的战后结果。
- 项目负责人确认 Phase 4 复盘后，才能进入 Phase 5。

### 9.3 实施顺序

1. **P4.1 持久军队与编军链**：建立军队实体，把将军库存与城市粮草真实转入军队，并支持驻城补充和解散归库。
2. **P4.2 路线行军与到达时间（已完成，2026-07-21）**：军队沿节点路线跨月移动，公开路径、出发地、目的地和预计到达月。
3. **P4.3 补给线与士气（已完成，2026-07-21）**：城市供给、距离、敌对路线和断粮按月改变粮草、兵员与士气。
4. **P4.4 遭遇、拦截、增援与撤退**：同节点敌军形成遭遇，邻近军队可以拦截或增援，败军按合法路线撤退。
5. **P4.5 持续围城**：军队抵达敌城后建立围城状态，支持围困、强攻、突围、等待援军与投降。
6. **P4.6 格子战回写与完整战争走查**：快速结算和真实格子战共用军队输入，统一回写幸存、补给、士气、英灵、撤退和占领。

## 10. Phase 5 — 世界危机与动态故事

**状态：未开始**
**阶段目标：用一条会改变地图压力的多阶段雪鬼主线，给战役建立中盘转折、共同威胁和终局高潮。**

### 10.1 计划范围

- 首先只实现雪鬼主线；电子世界危机后置为 Phase 8 复玩变体。
- 主线按预兆、边境压力、扩散、决战和余波推进，并根据玩家与 AI 选择改变城市、路线、事件和敌军。
- 世界主线成为可完成胜利路线，与统一/消灭和限时评议并存。
- 提供至少 3 条跨月事件链，能读取外交、战斗、占领、民心和世界记忆。
- 允许主要势力围绕危机合作、争功、背约或趁火扩张，并留下信誉后果。
- AI 能评估危机压力，在生存、主线和扩张目标之间调整计划。

### 10.2 退出门槛

- 雪鬼主线会在一场完整战役中至少两次改变地图局势或战略合法行动。
- 玩家可以通过主线贡献达成世界主线结局，不要求统一全部城市。
- 相同初始地图因关键选择至少产生两种可观察的主线分支和不同余波。
- AI 会参与、回避或利用危机，并通过可见目标解释其选择。
- 项目负责人确认 Phase 5 复盘后，才能进入 Phase 6。

## 11. Phase 6 — 圣物、祭坛与英灵战略

**状态：未开始**
**阶段目标：让圣物、祭坛和英灵的个人风险成为英灵城邦区别于普通 4X 的核心玩法。**

### 11.1 计划范围

- 实现圣物/祭坛实体的发现、保存、绑定、维护、释放、转移、夺取与损坏状态。
- 保留当前祭祀场随机召唤链；明确祭祀、城市绑定与圣物祭坛的关系，不用圣物替换已经确认的召唤规则。
- 为英灵增加最小忠诚、关系、个人任务、战略专长和危机反应；不把它们做成永久数值养成。
- 英灵沉睡、解绑定、城市失守、圣物夺取和流亡之间形成跨月风险。
- 实现圣物/祭坛胜利路线，并让其他势力能够侦察、阻止或反制。

### 11.2 退出门槛

- 圣物从发现到绑定、争夺或释放的完整生命周期可运行并可恢复存档。
- 英灵至少会因任职、战斗、城市失守、个人任务或主线选择产生一种持续战略后果。
- 圣物路线能够独立完成，但需要可被对手读取和干预的多月准备。
- 祭祀召唤、圣物祭坛与城市绑定规则没有互相矛盾的入口或文案。
- 项目负责人确认 Phase 6 复盘后，才能进入 Phase 7。

## 12. Phase 7 — 多人战役与产品硬化

**状态：未开始**
**阶段目标：让 2～4 名真人能够可靠完成长期战役，缺席、断线、升级或服务重启都不要求人工修档。**

### 12.1 计划范围

- 增加月度提交/准备状态、房主截止、缺席代理与 AI 临时托管，替代长战役中“所有真人同时在线才能推进”的唯一方案。
- 支持同势力多名真人分别任职、职位交接、撤换确认与空缺职位临时接管。
- 完成加入码撤销/重发、重连、身份恢复、战斗中断恢复和战役归档。
- 建立版本化存档迁移、自动备份、恢复演练、操作审计和管理修复工具。
- 完成正式数据库、HTTPS、错误/性能监控、限流、安全与隐私检查。
- 完成桌面、手机、键盘、触屏、色弱和减少动态效果的大战役全流程走查。

### 12.2 退出门槛

- 至少一次 2～4 人完整战役在有人缺席、重新连接或被临时托管的情况下正常结束。
- 服务重启和版本升级不会丢失已确认的战役行动或重复结算。
- 备份恢复、版本回滚、长期战役压力测试和关键故障手册完成。
- 战役关键路径满足上线版本的安全、隐私、可访问性和运维要求。
- 项目负责人确认 Phase 7 复盘后，才能进入 Phase 8。

## 13. Phase 8 — 复玩、平衡与运营

**状态：未开始**
**阶段目标：让第二场战役提出不同战略问题，并建立能持续判断节奏、路线和系统价值的数据能力。**

### 13.1 计划范围

- 增加电子世界危机、不同城邦诉求、资源/路线修正、主要势力阶段目标和战役种子变体。
- 建立战役完成率、月度流失、胜利路线使用率、和平/战争收益、AI 竞争力、经济与滚雪球看板。
- 把已经完成的账号武将熟练度接入战役角色经历、战役成就和复盘展示；不提供永久战斗数值加成。
- 恢复与大战役直接相关的挑战、好友邀请、观战和回放分享。
- 建立平衡节奏、规则版本、异常数据告警和内容配置流程。

### 13.2 退出门槛

- 不存在长期压倒其他路线且没有明确风险或反制的单一最优策略。

## 15. 每项开发任务的完成定义

一项面向玩家的功能只有同时满足以下条件才算完成：

3. 后端规则、前端预览/交互和玩家文案一致。
4. 手机端与桌面端均完成走查；适用时包含键盘和无障碍状态。
5. 已增加对应行为测试，并通过相关定向测试与完整测试。
6. 需要衡量的功能已经增加事件或指标，且不记录敏感凭据。
