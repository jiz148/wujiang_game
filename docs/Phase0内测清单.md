# Phase 0 内测清单

> 用途：验证“第一次好玩”的真实玩家退出门槛。工程走查不能替代本清单中的真实玩家样本。

## 0. 建立干净内测批次

不要直接使用日常开发产生过工程走查事件的 `var/analytics.sqlite3`。每轮真实内测先运行：

```powershell
python tools/phase0_playtest.py start p0-round-01 --host 0.0.0.0 --port 8000
```

工具会为该批次隔离分析库和测试账号库，并在 `var/phase0-playtests/p0-round-01/` 保存本地观察记录。已有批次默认拒绝覆盖；继续同一批次时显式增加 `--resume`。

每位玩家结束后，用匿名编号补记观察结果；不要在反馈中写姓名、联系方式、密码或房间凭据：

```powershell
python tools/phase0_playtest.py record p0-round-01 --sample desktop-01 --device desktop --browser "Chrome 1280x720" --completed yes --stuck-step completed --external-rules no --restart yes --feedback "能够独立完成"
```

随时生成本地复盘摘要：

```powershell
python tools/phase0_playtest.py report p0-round-01
```

摘要会同时列出设备样本覆盖、人工完成率、无需外部规则的完整路径、首次有效行动中位时间和自动埋点指标；它不会自动代替项目负责人确认。

## 1. 样本与设备

- 受试者不得在测试前阅读项目规则文档，也不要由开发者口头解释具体操作。
- 桌面端与手机端都要覆盖；建议首轮至少各 5 个有效样本，再根据失败集中点扩大样本。
- 有效样本必须从首页开始，并允许完整记录教学完成、退出或失败；开发者代操作不计入。
- 使用测试账号和非敏感昵称，不记录真实密码、联系方式或房间凭据。

## 2. 单人测试流程

1. 打开首页，询问玩家“你认为从哪里开始第一局？”但不给答案。
2. 玩家自行点击快速开始、完成登录/注册并进入固定教学。
3. 观察其是否能完成选中、移动、普攻、穿刺、结束回合和连锁响应。
4. 允许玩家按自己的方式完成自由战斗；不要提示规则。
5. 结算后要求玩家自行找到再次开始入口。
6. 记录设备、是否完成、退出步骤、主要困惑和是否查看了外部说明。

## 3. 每个样本记录

| 字段 | 说明 |
| --- | --- |
| 匿名样本编号 | 例如 `desktop-01`，不要使用姓名 |
| 设备 | 桌面 / 手机，附浏览器与大致视口 |
| 是否完成教学 | 是 / 否 |
| 首次有效行动时间 | 由 `/api/analytics/funnel` 汇总；个案可记录观察值 |
| 退出或卡住步骤 | `select_unit` 到 `win_objective` |
| 非法操作主要原因 | 范围、路径、目标、资源、未理解连锁等 |
| 是否打开外部规则 | 是则该样本不能证明“无需规则文档”门槛 |
| 是否找到再次开始 | 是 / 否 |
| 一句话反馈 | 玩家原意摘要，不记录个人身份 |

## 4. 工程走查结果（2026-07-14）

- 桌面 1280px：首页只显示三个主入口；快速开始未登录时显示统一账号门槛；登录后固定教学可进入。
- 桌面教学：步骤面板、固定棋子、移动行动、指定落点、最终站位预测和首次行动推进均实际点击验证。
- 手机 375×829：首页无整页横向溢出，三个入口为单列且触控高度约 45px。
- 手机教学：页面无整页横向溢出；教学面板和棋盘排在长武将详情之前；棋盘在自身容器内横向滚动；教学文字与按钮对比度已复查。
- 手机教学恢复：主动退出后首页同时提供“继续未完成教学”和“重新开始教学”；继续后回到同一房间及同一服务端步骤，页面仍无整页横向溢出。
- 聚合内测看板：桌面与手机视口均验证加载、空状态、刷新入口和响应式布局；看板只展示聚合指标。
- 浏览器控制台无错误或警告。

## 5. P0 退出判定

打开 `/analytics.html` 查看聚合内测看板（原始数据接口为 `/api/analytics/funnel`），重点读取 `metrics`：

- `first_effective_action_median_ms < 180000`。
- `tutorial_completion_rate >= 0.70`；样本不足时不得标记达标。
- `match_completion_rate`、`invalid_action_rate`、教学与对局时长用于定位稳定性和理解问题。

此外必须确认：测试玩家无需外部规则完成 AI 教学、胜利后能再次开始、相关行为测试与完整测试通过。完成数据复盘后仍需项目负责人明确确认，才能退出 Phase 0。
