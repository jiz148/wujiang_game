- Phase 3 P3.2 neutral diplomacy is implemented in `src/wujiang/strategy/diplomacy.py`. A major-faction lord may spend 1 command on one replaceable ordinary negotiation per adjacent active neutral per month: aid transfers 80 food/60 money for +18 relation; trade exchanges 40 money for 80 city food and +8; accepted protection transfers 40 money/60 troops for +15 and a structured agreement; accepted non-aggression gives +6 and an agreement; tribute and intimidation resolve from actor border strength or prior intimidation with visible acceptance/refusal effects. - Phase 3 P3.3 promise and reputation lifecycle is implemented across `src/wujiang/strategy/diplomacy.py`, `models.py`, `simulation.py`, and `neutral_city_states.py`. Major factions persist 0–100 diplomatic reputation starting at 50; aid/fulfillment reward it, coercion/incitement/breach/failure reduce it, and reputation below 30 blocks new protection/non-aggression with a public reason. - Phase 4 P4.1 persistent armies and form-up are implemented in `src/wujiang/strategy/armies.py`, `models.py`, office authorization, monthly queue settlement and the General workspace. Each active General can command at most one non-disbanded army with an exact unit inventory, troop-derived manpower, supply/capacity, morale, location, status and order. - Phase 4 P4.2 route movement and ETA are implemented in `src/wujiang/strategy/armies.py`, `models.py`, `simulation.py`, queued action settlement and the General/map workspace. Every connected node edge costs one strategic month. `set_army_movement` costs 1 command, uses the army id as its replacement key, and supports deterministic shortest-path march/reroute plus halt. Armies persist origin/destination/full route/progress/departure/ETA; month advancement moves each marching army one edge and moves its commander hero with it. Arrival or halt at an owned city yields `garrisoned`, otherwise `deployed`. Supply/morale loss, encounters and siege are deliberately not resolved before P4.3–P4.5. The map shows army badges and highlights remaining route edges; SVG route classes must be applied with `setAttribute`, not `className`.
- Phase 4 P4.3 supply/morale is implemented in `src/wujiang/strategy/armies.py`, `models.py`, monthly simulation, queued actions and General/map surfaces. After monthly movement, need is `max(10, ceil(manpower / 20)) + 5 per supply-route edge + 5 when marching that month`. A deterministic best owned city supplies through owned intermediate city nodes; distance 0 is local, 1–2 edges open, 3+ strained at half delivery, and a neutral/enemy intermediate node severs delivery. Transport consumes real city food. Full supply restores/holds morale, severed reserves cost 3 morale, shortage costs 12 and increments starvation. Two consecutive shortage months remove the cheapest real registered unit; an empty army becomes `destroyed`. `load_army_supply` lets a General spend 1 command to load food only at the army's current owned garrison city. The map and workspace expose source, route, distance, need, last transfer/consumption and starvation risk.

- Strategy office authority is implemented in `src/wujiang/strategy/offices.py`. The fixed v0.1 offices are lord, grand general, general, and governor; players and AI act through offices, never directly through a faction. - Durable default conventions: do not ask again when these apply. If an active skill has no cost text, it costs 0 mana; if any skill has no use limit/cooldown text, it has no use-count limit; if a damaging skill/area has no fixed `伤 n` and no `没有伤害`, use current attack; if no `破魔` text, it does not pierce shields; if no ally/enemy qualifier, it affects both sides; `单位` includes heroes, summons, and clones; `武将` means non-summon non-clone hero body only; stat reductions to attack/defense/speed/range floor at 1; modifying the `mana` stat means both max mana and current mana are modified then clamped.

- Durable extra-move-count rule: `每回合移动次数 +1` grants one extra normal-move action per turn; each normal move is still separately capped by current speed.
- Durable round-duration rule: if docs/rules say an integer `持续 N 轮` for a unit status or integer-round local effect, count it on the affected unit's or owner's own round boundary, not on every global hero turn. Half-round effects such as `1.5轮` / `2.5轮` still need explicit per-skill handling.
- Durable passive-count rule for n-v-n: passive/reaction skills written as `每回合最多 X 次` reset separately in each opposing hero turn, not once per whole round. - Durable classic spawn rule: classic board size and fixed spawn slots depend on both roster size and opening footprint sizes, including mounted-entry or future oversized heroes. Do not hard-code classic mode to 8x8 or assume only 1x1 / 1x2 / 2x2 entries. Spawn slots are assigned by the precomputed classic turn order, not by lobby pick order.
- Durable random spawn rule: `random` reuses the same board-size calculation and interleaved turn-order rules as `classic`, including opening footprint sizing for mounted-entry or future oversized heroes, but replaces fixed symmetric spawns with random per-side spawns inside fair left/right bands.
- Summon default stats: omitted summon stats default to attack 1, defense 1, speed 1, range 1, mana 0.
- Knight durable rule: the first summon in a Knight hero's skill text is the mount unless explicitly overridden. Default mount footprint is vertical `1*2`, and mounts default to the `可乘骑` trait.
- Knight durable rule: Knight heroes enter battle already having summoned their mount and already mounted unless the hero text overrides that. A rider can have only one own mount on the field at once; when that mount is destroyed, the rider must wait through one own turn before summoning it again.
- `可乘骑` durable rule: if mounted unit `a` is ridden by unit `b`, only `a` receives damage and skill effects; when `a` is threatened, `b` can still chain for `a`, and support reactions such as Protection or Knockback resolve on `a`. Mounted state ends when `a` uses a normal move or movement skill to leave `b`.
- `可乘骑` also redirects point targeting: clicking rider `b` still causes the damage or skill effect to resolve on mounted unit `a`.
- Mounted movement durable rule: when mounted unit `a` moves, rider `b` is carried along. Rider `b` may still move independently; if `b` leaves `a`'s occupied cells, mounted state ends immediately.
- Mounted AI durable rule: AI should not voluntarily choose rider-body normal movement or generic rider movement skills while mounted, because that dismounts and splits the knight from the mount. It should keep the rider fighting from the mount and use mount movement for repositioning. ## Testing Policy

- Add focused tests for every durable gameplay rule change.
- After every gameplay, room-flow, or player-visible UI change, add or update at least one behavior scenario that exercises the changed feature from the player's point of view.
- Prefer `tests/test_behavior.py` for scenario-style coverage that crosses room APIs, turn flow, chain windows, targeting, or frontend-facing state payloads.
- Use direct `DamageContext` tests only for engine-level behavior; prefer `battle.perform_action(...)` for player-facing skill behavior.
- Run targeted tests for the changed rule first.
- Run the relevant behavior scenarios after targeted tests.
- Run `python -m unittest discover -s tests` before final response when feasible.
- `git diff --check` may show LF/CRLF warnings on this Windows repo; treat actual whitespace errors separately from those warnings.
